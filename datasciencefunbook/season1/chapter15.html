🌙 Data Science: Alerts at 3AM and Other Joys of Deployment

Season 1 — Episode 15

🎙️ Original Episode

The model is live. The pipeline is humming. The dashboard looks sharp.

Then you get a Slack ping at 3:12AM.

“KPI looks weird. Can you check?”

Welcome to production.

The joys include:

  • Unmonitored edge cases
  • Silent schema breaks
  • Random timezone bugs
  • “Hotfixes” in the notebook

Deployment isn’t the end. It’s the start of reality.

✨ Build for it.

🔍 Let’s Go Deeper…

Putting a model into production is not the finish line. It’s where all the real issues begin.

Deployments introduce operational concerns: uptime, latency, versioning, logging, security, cost control — and people using it in unexpected ways.

🛎️ Common Post-Deployment Surprises

  • Model assumptions violated by real-world inputs
  • Unexpected nulls or missing features
  • Changes to upstream data pipelines
  • Sudden shifts in input distributions
  • Infra quota limits (API, storage, compute)

💻 Hands-on – Simulating a Drift Alert Trigger

import numpy as np

# Simulate input feature distribution
baseline = np.random.normal(loc=50, scale=5, size=1000)
live_input = np.random.normal(loc=65, scale=5, size=1000)

drift_detected = abs(baseline.mean() - live_input.mean()) > 10

print("🚨 Drift detected?" , drift_detected)

✅ Lessons Learned

  • Shipping the model is the start of a new lifecycle — not the end of the project.
  • Monitoring and alerting are part of the ML product.
  • Expect that your model will break — plan for recovery, not perfection.

⚠️ Common Mistakes

  • Treating deployment as an afterthought
  • No rollback strategy or alert routing
  • Relying on one person to be ‘on-call’ for everything

🎯 Practice Challenge

  1. Identify a deployed model or system (real or hypothetical)
  2. Create a drift test or simple input validation check
  3. Define alert criteria and notification channel
  4. Document how your team should respond if triggered

📎 Tools & Resources

  • Seldon Core, BentoML, FastAPI for ML deployment
  • Evidently AI and Prometheus for post-deploy monitoring
  • Book: “Building Machine Learning Powered Applications” by Emmanuel Ameisen

🧠 Final Thought

You don’t need to panic every time an alert goes off.
But if no alert goes off — you should be worried.
